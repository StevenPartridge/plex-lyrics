Plex Lyrics PiP

Chrome + Firefox Extension In Development

Small WebExtension that mirrors Plex Web lyrics into a local Picture-in-Picture window while keeping lyric text session-only.

The extension now has one product mode: the user opens a focused Lyrics PiP window from inside Plex Web. While PiP is open, pip-page.js keeps Plex's own lyrics panel available as the source and mirrors only the current page's lyric DOM into the PiP window. When PiP closes, the extension stops forcing Plex lyrics open.

The older screenshots in images/ and wiki/assets/ show pre-PiP or popup-era UI and need refreshing after a live Plex PiP smoke test.

User Flow

Open Plex Web

The public marketplace content script loads on https://app.plex.tv/*, then verifies the page looks like Plex before adding UI.

Click Lyrics PiP

pip-page.js injects a small Lyrics PiP button in the Plex page. The trusted page click calls window.documentPictureInPicture.requestWindow().

Mirror Plex lyrics

While PiP is open, the page-world script snapshots Plex's visible lyrics DOM, active-line class, no-lyrics state, load-error state, now-playing metadata, and media progress in memory only.

Keep the source panel available

If Plex closes its lyrics panel during a song change, PiP throttles a click on Plex's main lyrics button so the PiP window has a current source to mirror.

Close PiP to stop

Closing the PiP window clears the PiP runtime state. Plex lyrics are no longer forced open until the user clicks Lyrics PiP again.

Core Selector

The integration depends on Plex exposing data-testid="lyricsButton" in the player UI.

Visible State

Lyrics are considered open from ARIA selected/pressed attributes, data-selected, or the legacy Plex selected class Link-isSelected-x0P_By.

Operation

Local Load

  1. Open Firefox and navigate to about:debugging.
  2. Choose This Firefox.
  3. Choose Load Temporary Add-on.
  4. Select this repository's manifest.json.
  5. Open Plex Web, play music, click Lyrics PiP, and verify the PiP window mirrors lyrics.

Risks

RiskWhy It MattersLikely Fix
Plex DOM driftThe selector and selected-state class are implementation details in Plex Web and may change without notice.Smoke test against current Plex; add fallback selectors if the UI changed.
Document PiP supportThe feature depends on browser support for arbitrary HTML Picture-in-Picture from a trusted click.Keep feature detection and unsupported-browser copy; test Zen, Firefox, Chrome, and Edge separately.
Page-world content script supportZen/Firefox behavior depends on the manifest world: "MAIN" script path working reliably.Continue temporary-load smoke tests before public packaging.
PiP toast placement driftPlex layout changes could make PiP unavailable/error toasts feel poorly positioned.Keep toast fixed and minimal; adjust after live player screenshots.
Self-hosted Plex supportThe public marketplace manifest no longer injects into arbitrary /web/ paths, which avoids broad site access but drops direct self-hosted URL support.Use app.plex.tv for marketplace release; revisit self-hosted support later with an opt-in permission flow.
