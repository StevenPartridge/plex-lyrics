Plex Lyrics Companion

Firefox Add-on In Development

Small WebExtension that keeps Plex Music lyrics available while respecting manual close and showing clear session state.

Overview

StatusMV3 overhaul in progress locally. Published store listing may still describe the older Auto Clicker version.

The extension exists to work around the friction of Plex not automatically keeping lyrics visible across track changes. The user starts a session from a Plex-native popup. The content script observes Plex Music, opens lyrics when appropriate, and pauses automation immediately when lyrics are manually closed.

The older screenshots in images/ and wiki/assets/ show the pre-overhaul timer UI and need refreshing after a live Plex smoke test.

User Flow

Open the extension popup

popup.html renders a Plex-native control center with current state, Plex tab status, lyrics status, primary action, Stop, Open/Focus Plex, Check again, and last event.

Restore saved session state

popup.js asks the background script for the stored lyricsCompanionSession, then renders state-specific copy and actions.

Start a session

The popup sends START_SESSION. background.js stores a session state, updates the toolbar badge, and asks matching Plex tabs to sync.

Observe Plex

content.js uses a MutationObserver and a slower fallback poll to watch the lyrics button and selected state.

Click lyrics when needed

When active and lyrics are hidden without a trusted manual close event, the content script dispatches a Plex-style click sequence on the lyrics button and returns explicit statuses.

Pause on manual close

Trusted user interaction with the lyrics control or likely close control followed by lyrics hiding enters pausedByManualClose. The badge shows PAUSE, and lyrics are not reopened until Resume.

Core Selector

The integration depends on Plex exposing data-testid="lyricsButton" in the player UI.

Visible State

Lyrics are considered open from ARIA selected/pressed attributes, data-selected, or the legacy Plex selected class Link-isSelected-x0P_By.

Operation

Local Load

  1. Open Firefox and navigate to about:debugging.
  2. Choose This Firefox.
  3. Choose Load Temporary Add-on.
  4. Select this repository's manifest.json.
  5. Open Plex Web, play music, start the add-on, and verify lyrics open after a track change.

Risks

RiskWhy It MattersLikely Fix
Plex DOM driftThe selector and selected-state class are implementation details in Plex Web and may change without notice.Smoke test against current Plex; add fallback selectors if the UI changed.
MV3 cross-browser behaviorFirefox uses MV3 background scripts while Chrome uses service_worker; both entries point at the same file.Manual load in Firefox; optional later package validation with web-ext or browser-specific smoke tests.
Manual close detection scopeThe close detection depends on trusted interactions with likely lyrics or close controls before the selected state disappears.Smoke test live Plex; add selectors for any close affordance the observer misses.
Toast placement driftPlex layout changes could make the toast feel poorly positioned.Keep toast fixed and minimal; adjust after live player screenshots.
Broad content match*://*/web/* supports self-hosted Plex, but also matches unrelated pages with /web/ paths.Runtime guards check for likely Plex pages before interacting.
