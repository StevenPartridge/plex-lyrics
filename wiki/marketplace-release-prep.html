Marketplace Release Prep

Release Packet Chrome + Firefox

Pre-submission audit, package scope, privacy posture, reviewer instructions, and store listing copy for publishing Plex Lyrics PiP.

Verdict

The codebase is close to marketplace-ready after the PiP-only pivot. The active extension is single-purpose, MV3, reviewable plain JavaScript, and does not fetch, transmit, or persist lyric bodies. The biggest release-risk item was the broad *://*/web/* content match for self-hosted Plex. The marketplace manifest now targets only https://app.plex.tv/*.

AreaStatusNotes
Product focusReadyOne purpose: show Plex Web lyrics in a local Picture-in-Picture window.
PermissionsTightenedNo extension API permissions, no static host_permissions key, one content-script match for https://app.plex.tv/*.
Data handlingLow riskNo analytics, no network calls, no lyric storage. Only a local PiP viewport-size preference is saved.
Browser supportMust verifyChrome 116+ and Firefox 151+ minimums reflect Document PiP support; run live smoke tests before submission.
AssetsNeeds refreshExisting screenshots are from older UI generations. New store screenshots should use fake lyric text.
This is not legal advice. The release posture minimizes content-rights and trademark risk, but a public listing should still avoid implying Plex or lyric-provider affiliation.

Policy Audit

The store posture should lead with narrow purpose, narrow access, transparent behavior, and no data transmission. The implementation now aligns with that story.

Marketplace ConcernCurrent StateSubmission Notes
Chrome single purpose All features directly support one Plex lyric PiP experience. Describe previous/play-pause/next controls as part of the PiP lyric viewer, not a general Plex remote.
Chrome permissions No extension API permissions. The content script runs only on https://app.plex.tv/*. Justification: the page access is required to read the lyrics Plex already renders and open Document PiP from a trusted page click.
Chrome MV3 code No remote code, no eval(), no external scripts, no obfuscation, no build bundling. Submit the plain source package generated by the script; reviewer can read behavior directly.
Chrome IP/impersonation Name uses Plex descriptively and no Plex logos are bundled. Listing must say "not affiliated with or endorsed by Plex" and avoid official marks.
Firefox data transmission No data transmission occurs. AMO data disclosure should state no collection/transmission. If future analytics are added, consent flow is required.
Firefox reviewability Plain JavaScript files, no minification, no dependencies, no generated runtime code. Include reviewer notes explaining the need for world: "MAIN" and Document PiP.
Do not submit screenshots or promotional video that shows real copyrighted lyric excerpts. Use placeholder lyric lines made for the listing.

API Audit

The extension is intentionally light on extension APIs. Its risky browser dependency is not a privileged extension API; it is the web-platform Document Picture-in-Picture API exposed to the Plex page.

API / CapabilityUseRisk
Document Picture-in-PictureOpens the always-on-top lyrics window from the in-page click.Limited browser support; requires a secure context and transient user activation.
MutationObserverWatches Plex UI changes so lyrics and active-line state update.Performance-sensitive; mitigated by throttled rendering and narrow app.plex.tv match.
Media Session metadataOptional title/artist display inside PiP.Session-only display. Do not persist listening history.

localStorageSaves only PiP viewport {width,height} on the Plex origin.Safe but disclose as local UI preference. Legacy storage key is migrated.

Privacy

Use this as the base privacy-policy text for both marketplaces and the README.

Privacy policy draft: Plex Lyrics PiP does not collect, sell, share, or transmit personal data. The extension does not fetch lyrics from external services and does not send lyric text, track titles, artist names, Plex server information, browsing history, or account information anywhere. Lyrics and playback metadata are read only from the current Plex Web page and displayed locally in the Picture-in-Picture window. The extension stores only the last Picture-in-Picture viewport size as a local UI preference on the device.
Data TypeHandling
Lyric textRead from the current Plex page and rendered in the PiP window. Not stored, logged, transmitted, or exported.
Track title / artistRead from browser Media Session when available. Displayed locally only.
Plex account or server dataNot read intentionally and not transmitted.
PiP size preferenceStored locally as width and height numbers so the next PiP opens at the user's preferred size.

Store Listing

Recommended Name

Plex Lyrics PiP

Short Description

Show Plex Web lyrics in a local Picture-in-Picture window.

Long Description

Plex Lyrics PiP adds a small Lyrics PiP button inside Plex Web. When you click it, the extension opens a local Picture-in-Picture window and mirrors the lyrics Plex already shows for the current track.

It works with timed and plain lyrics, keeps the Plex lyrics panel available while PiP is open, and shows a calm state when Plex has no lyrics or cannot load them. The PiP window also includes compact previous, play/pause, next, now-playing, and progress controls that use Plex's own player controls.

The extension does not provide, fetch, scrape, store, or share lyrics. It only displays lyrics already available in your Plex Web session. Plex Pass, region, and track availability still determine whether lyrics appear.

Plex Lyrics PiP is an independent extension and is not affiliated with, endorsed by, or produced by Plex.

Feature Bullets

  • Portrait Picture-in-Picture lyric viewer for Plex Web.
  • Mirrors Plex-provided timed or plain lyrics.
  • Highlights Plex's current lyric line when timed lyrics are available.
  • Shows no-lyrics and Plex lyric-load-error states clearly.
  • Includes compact playback controls and local progress readout.
  • No analytics, external lyric lookup, or lyric storage.

Reviewer Notes

Use this in the Chrome "review notes" and AMO reviewer notes fields.

Reviewer instructions draft: This extension works on https://app.plex.tv/*. Sign in to Plex Web, start music playback, open the Plex lyrics panel for a track with lyrics, then click the injected Lyrics PiP button near the bottom-right of the page. The extension opens a Document Picture-in-Picture window from that user click and mirrors only the lyrics Plex already displays in the page. It does not make network requests, fetch lyrics, use analytics, or transmit data. If the test account or track has no lyrics, the PiP window shows a local "No lyrics for this song" state.
QuestionAnswer for Reviewers
Why page-world script?Document PiP access must stay in the page realm in Firefox/Zen to avoid cross-origin access failures when working with the returned PiP window.
Why app.plex.tv access?The extension must read the visible Plex lyrics DOM and click Plex's own lyrics button only while PiP is open.
What is stored?Only the last PiP viewport width and height. No lyric text, track metadata, account data, or server data is stored.
Does it bypass Plex Pass or lyric availability?No. It mirrors only lyrics Plex Web already displays to the signed-in user.
